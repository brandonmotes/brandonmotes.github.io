I began working for this lab in the summer of 2018 and have continued to work in this lab. I develop automated test beds for perovskite solar panels produced in the lab to guide development.

Summer 2018

When I joined the lab, the team had already set up a time resolved photoluminescence (TRPL) test bed, an optical test that can be used to judge efficiency. This setup required a good deal of manual work as the user needed to use a range of optical filters requiring the user to stop, change the filter by hand, and then begin another run, consequently generating seperate files that must be tracked and kept together. Furthermore, the light sensor was extremely sensitive and could only be operated in the dark. If the lid was taken off while the sesnor was on, it would be damaged. My work over the summer was automating as much of this work as possible.

I started with creating a stepper motor driven wheel to hold the filters, so the user would not have to be changed them by hand. I printed most of the assembly out of PLA on a 3D printer, and made it mountable to a standard optical breadboard post for easy placement.

Next I completed a current project they had begun and integrated a reed switch on the lid that would trigger a shutter covering the light sensor to protect it form overexposure.

With all of the parts being controlled by software now (all the other optical parts could be conrolled with commercial software), I had to merge the control of all of the components. The chosen solution was to write completely new custom software. This would also allow future features to be easily integrated. Most of the summer was spent developing this software.

The software was built off of an open source python project called SourceFoundry. This project was used to provide the GUI for the project. I wrote a new backend for the program to interface all of the hardware compoennts. The hardware components I built were connected to a RaspberyPi that was then connected to the LAN. The lab computer could then send commands to the RaspberryPi via a TCP port. The commercial hardware was either either connected to a PCI slot and given commands via a DLL file.

The final product allowed the user to put in a single sample, set the intensity regimes, collection times, and other important parameters, then click run once and come back to a single data file. It also provides the extra layer of protection for the optical sensor.

Spring 2019

After finsihing the first prototype, the lab wanted a version that could perform these characterizations on multiple samples, a sheet, or a roll in-situ during fabrication. This presented an entire new regime of motor control to the project. I created a CAD model of a desgin and began construction.
